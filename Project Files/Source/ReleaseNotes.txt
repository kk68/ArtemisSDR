# ArtemisSDR

A native SDR host for the SunSDR2 DX, dedicated to the Artemis II mission. Forked from Thetis by Richard Samphire (MW0LGE), itself descended from OpenHPSDR (Doug Wigley, W5WC) and PowerSDR (FlexRadio Systems). Released under GPL v2.

Report issues at: https://github.com/kk68/ArtemisSDR/issues

---

## Beta v1.0.1 — First public release

First ArtemisSDR release under its own name and version line. Speaks the SunSDR2 DX's native UDP protocol directly, with no ExpertSDR proxy, bridge, or firmware changes.

### Receive
- RX1 with panadapter, waterfall, and audio on every mode ArtemisSDR supports
- RX2 as an independent second receiver with its own VFO B and audio path
- RX antenna selection (primary / auxiliary inputs)
- Live firmware version and radio serial number display in the title bar and Setup → H/W Select

### Transmit
- MOX and TUNE via native wire protocol — reliable across consecutive attempts (11/11 TUNE and 10/10 voice MOX verified)
- TUNE in every mode lands on the dial frequency: SSB, CW, AM, FM
- Voice SSB, AM, CW, and digital modes transmit on the correct sideband / carrier
- External PA (xPA) control during MOX and TUNE
- TX power calibrated linearly on 40 m — drive slider in watts maps to actual RF within ~1 W across 10-90 W
- TX antenna selection (primary / auxiliary outputs)
- AM Carrier Level setting wired through to the WDSP AM modulator

### General
- Power-on to RX in ~1-1.5 seconds, comparable to ExpertSDR3
- Sub-second band switching, native protocol, no session teardown
- All of the WDSP DSP stack: NR, NR4, ANF, NB/NB2, EQ, CESSB, CFC, notch, compander
- VAC audio routing (CABLE, VoiceMeeter, etc.) works on both RX and TX
- Clean power-off / power-on cycling from the ArtemisSDR UI
- Proper `PA Gain By Band` and per-drive offsets integration — calibrate per band the same way you'd calibrate any Thetis-family radio

### System integration — new in this release
- Isolated user data under `%AppData%\ArtemisSDR\` — does not share DB, settings, logs, or skins with any Thetis-for-ANAN install
- Dedicated single-instance mutex — ArtemisSDR and Thetis can run on the same machine without mutual detection
- Windows Firewall rules registered under the ArtemisSDR name
- ArtemisSDR-branded registry paths, Start Menu shortcut, desktop shortcut, and installer

### Known limitations
- **TX power calibration**: 40 m is locked; other bands fall back to the 40 m curve. Expect a few dB deviation until you run a per-band sweep (see README → "TX power calibration per band")
- **Fwd Pwr meter**: not yet wired to the SunSDR telemetry stream. Use an external wattmeter. Planned for the next release.
- **PS-A, 2-TONE, DUP**: grayed out on SunSDR — these depend on a feedback-loop path the radio doesn't expose. Hardware-architecture constraint, not a bug.
- **Diversity mode**: unsupported. RX2 follows RX1's antenna selection.
- **MON / DUP audio routing**: not fully settled during TX. If you need to monitor your own transmission, use a second receiver.
- **Occasional post-TX raspy audio**: intermittent; cycling VAC clears it. Tracked as a polish item.

### Versioning scheme

- **Patch** (1.0.1 → 1.0.2): routine fixes and small features
- **Minor** (1.0.x → 1.1.0): meaningful new feature, batch of related fixes
- **Major** (1.x.y → 2.0.0): architectural change, protocol overhaul, or exit from Beta

### Installation notes

If you are upgrading from a pre-release build of this fork, your previous database location was `%AppData%\OpenHPSDR\Thetis-x64\` — that data is NOT migrated automatically. If you had settings you want to preserve, copy them manually before first launch.

Hardware model selection in Setup is SUNSDR2-DX only. Other Thetis-supported radios (ANAN / Hermes / Orion / Saturn) are hidden from the UI; the underlying code paths remain in place but are not exercised by this build.

## Upstream Thetis heritage

ArtemisSDR's DSP stack, panadapter, CAT server, TCI server, metering architecture, and the entire Console framework come directly from Thetis by Richard Samphire (MW0LGE) — which itself traces back through Doug Wigley's OpenHPSDR Thetis fork and, ultimately, to FlexRadio's PowerSDR. Decades of prior work live inside this codebase.
